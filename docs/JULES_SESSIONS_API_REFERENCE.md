# Jules Sessions API Reference

Detailed documentation for the Jules Sessions API resource and its methods.

**Base URL**: `https://jules.googleapis.com`
**Reference Documentation**: [Google Developers - Jules Sessions](https://developers.google.com/jules/api/reference/rest/v1alpha/sessions)

---

## 1. Resource: Session

A session represents a single task or interaction with Jules.

### Fields
| Field | Type | Description |
| :--- | :--- | :--- |
| `name` | `string` | Output only. The resource name of the session. Format: `sessions/{session_id}` |
| `title` | `string` | Required. The title of the session. |
| `prompt` | `string` | Required. The initial instruction for the agent. |
| `sourceContext` | `object (SourceContext)` | Required. Context for the source code to be used in the session. |
| `automationMode` | `enum (AutomationMode)` | The automation mode of the session. |
| `state` | `enum (State)` | Output only. The current state of the session. |
| `url` | `string` | Output only. The web app URL to view the session. |
| `outputs` | `[]object (SessionOutput)` | Output only. Resulting artifacts (e.g., Pull Requests). |
| `createTime` | `timestamp` | Output only. When the session was created. |
| `updateTime` | `timestamp` | Output only. When the session was last updated. |

---

## 2. Methods

### `create`
Start a new coding task.
- **HTTP Request**: `POST /v1alpha/sessions`
- **Request Body**: `Session` object.

### `get`
Retrieve details of an existing session.
- **HTTP Request**: `GET /v1alpha/sessions/{session_id}`

### `list`
List sessions for the authenticated user.
- **HTTP Request**: `GET /v1alpha/sessions`
- **Query Parameters**: `pageSize` (integer), `pageToken` (string).

### `approvePlan`
Approve a plan generated by the agent.
- **HTTP Request**: `POST /v1alpha/sessions/{session_id}:approvePlan`

### `sendMessage`
Send a follow-up message to an active session.
- **HTTP Request**: `POST /v1alpha/sessions/{session_id}:sendMessage`
- **Request Body**: `{ "message": "string" }`

## 3. Data Structures

### SourceContext
```json
{
  "source": "string", 
  "githubRepoContext": { 
    "startingBranch": "string" 
  }
}
```
- `source`: Required. Format: `sources/{source_id}`.
- `githubRepoContext`: Optional. Context for GitHub repositories.

### AutomationMode
- `AUTOMATION_MODE_UNSPECIFIED`: Default.
- `AUTO_CREATE_PR`: Automatically create a PR when the task is complete.

### State
- `QUEUED`: Waiting for resources.
- `PLANNING`: Agent is analyzing the task.
- `AWAITING_PLAN_APPROVAL`: Waiting for user to call `:approvePlan`.
- `AWAITING_USER_FEEDBACK`: Waiting for user response via `:sendMessage`.
- `IN_PROGRESS`: Executing the task.
- `PAUSED`: Execution paused.
- `FAILED`: Task ended with an error.
- `COMPLETED`: Task finished successfully.

### SessionOutput
Contains the result of the session.
- `pullRequest`: `object (PullRequest)`

### PullRequest
```json
{
  "url": "string",
  "title": "string",
  "description": "string"
}
```

## 4. Resource: Activity

Activities are events within a session (messages, plans, progress updates).

### Fields
| Field | Type | Description |
| :--- | :--- | :--- |
| `name` | `string` | Format: `sessions/{session_id}/activities/{activity_id}` |
| `originator` | `string` | Entity that created activity: `user`, `agent`, `system`. |
| `createTime` | `timestamp` | When the activity occurred. |
| `artifacts[]` | `[]object (Artifact)` | Data produced (patches, media, logs). |
| `agentMessaged` | `object (AgentMessaged)` | Union: Agent message content. |
| `userMessaged` | `object (UserMessaged)` | Union: User message content. |
| `planGenerated` | `object (PlanGenerated)` | Union: Details of a new plan. |
| `planApproved` | `object (PlanApproved)` | Union: Confirmation of plan approval. |
| `progressUpdated` | `object (ProgressUpdated)` | Union: Status update. |
| `sessionFailed` | `object (SessionFailed)` | Union: Failure reason. |

### Activities Methods
- `list`: `GET /v1alpha/sessions/{session_id}/activities`
- `get`: `GET /v1alpha/sessions/{session_id}/activities/{activity_id}`

---

## 5. Artifacts and Plans

### Artifact
- `changeSet`: Code changes via `GitPatch`.
- `media`: Images/Videos produced.
- `bashOutput`: Logs from executed commands.

### Plan
- `id`: Unique identifier for the plan.
- `steps[]`: Array of `PlanStep` (id, title, description, index).
- `createTime`: When the plan was proposed.
